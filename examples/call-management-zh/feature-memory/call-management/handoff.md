# 通话管理交接

## 当前目标

在主要只读能力已经实现后，使用真实数据验证通话管理模块。

## 当前状态

- 工作状态：`VERIFYING`
- 记忆状态：`VERIFYING`
- 通话列表、详情抽屉、事件时间线、转写展示、质检结果、录音播放和 WebSocket 刷新已实现。
- 录音播放问题已收敛，记录在 `debug/BUG-20260609-001-recording-playback-auth/`。

## 新 Agent 必须知道

- 当前 feature 明确只做只读观测。
- 列表接口应保持轻量，不加载大 JSON、转写、质检结果或音频内容。
- 详情、事件、转写、质检和录音都按需获取。
- 录音播放必须通过带鉴权的后端代理。
- 浏览器原生 `<audio>` 或 `window.open` 请求可能不会携带鉴权头；应使用带鉴权的 HTTP client 获取 blob，再创建 object URL。
- 手机号、对象存储 key、Token、Cookie 和生产日志必须脱敏。

## 推荐下一步

1. 使用真实数据验证列表筛选和分页。
2. 使用包含转写、质检和录音元数据的通话验证详情抽屉。
3. 验证 WebSocket 在有效时间窗内刷新，在无效时间窗内停止。
4. 确认录音播放和下载仍使用带鉴权的 blob 流程。
