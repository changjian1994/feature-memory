# 通话管理

## 目标

在管理后台新增一个只读的通话管理模块，帮助运营和研发查看 AI 外呼通话会话。

## 范围

- 通话记录列表和筛选。
- 通话详情抽屉。
- 事件时间线。
- 转写和质检结果展示。
- 通过带鉴权的后端代理播放和下载录音。
- 列表和详情的轻量 WebSocket 刷新。

## 非目标

- 挂断通话。
- 重试通话。
- 数据修复或补偿。
- 在管理后台保存原始音频。
- 在 memory 中记录生产日志原文、Token、Cookie、对象存储 key 或客户数据。

## 成功标准

- 新 Agent 可以通过 `ai-handoff.md`、`index.md` 和本 feature 的 `handoff.md` 快速接手。
- 大字段按需加载，不进入列表分页查询。
- 录音播放不暴露对象存储身份或直连 URL。
- 关键设计取舍和已解决 Bug 能在后续任务中复用。
